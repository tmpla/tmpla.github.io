バージョン管理システムと言う言葉があって、これは何かというとファイルの履歴をとっておいて後でどんな感じに変更したか、とかその時点まで戻せる、みたいなことが出来るもののことである。（簡単に言えば）

エクスプローラでShiftキーを押しながらDeleteキーで削除しちゃったり、rm -rf fileしちゃったら基本的にはそれを元に戻すのは非常に難しいのだけれども、これを戻す事が出来ます。
また、同じファイルを複数人の人が一斉に編集する場合、例えばExcelの共有モードで共有サーバーに置いてやるとそのうち破綻するが、編集内容を単純な内容ならうまくマージしてくれたり、編集内容が競合してたらどこが競合しているかをなるべく単純に判断出来るようにマークしてくれたりもする。
（バイナリフォーマットはさすがに難しいところがあるが）

プログラム開発工程では特にテキストファイル編集の応酬なので、こういうツールは非常に便利でありそれなりの規模のプロジェクトでは普通使っているものである。

バージョン管理システムは前述の通りないと困っちゃうレベルのツールになりつつあるので、それなりに歴史もある。
古くはdiffというテキストファイルの差分がわかるツールを使ってテキストデータを切り貼りしていたりしたのだが、
そのうちCVSやVisual Sourcesafe等のツールが開発され、Subversionで爆発的に普及した感じで自分の中ではクライマックスを迎えていた。

しかし、githubというのでなんだかオープンソースのコードを公開するのが流行りだして、gitというのがあるのを知った。
githubからこれを使う時には

git clone project

とやるので、最初はクローン？？とか、ローカルリポジトリ？？とかSubversion脳にはよくわからない概念があったのだが
Subversionを使う場合よりも何より良いのが「気軽に」リポジトリを作成してバージョン管理できてしまうのが良くて最近結構使うようになっている。
gitの競合？ツールには、Mercurial、Bazaarというのがあり、これをバージョン管理の新・御三家と呼ぶ（自分の中では）
