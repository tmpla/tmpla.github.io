以前にandroid開発環境の構築の手順を書きましたが、
最近また環境構築してみたら全然簡単になっていたので記念にメモしておきます。

ちなみに以前の記事の際は多分ADTのバージョンは10.0くらい。今回は16.0.1です。
さらに、今回の構築環境は基本はMacなのですが、Windows7でやっても手順あんまりかわらないようでした。それほどシンプル。

ADTに関してはいろんな人がブログに書いているようなのですが、それぞれ個性があるので基本は下記を参照した方が良いです。

ADT Plugin for Eclipse | Android Developers
http://developer.android.com/intl/ja/sdk/eclipse-adt.html

ベースの環境はMacbook Air(MC965J/A)で、下記コマンドの通りです。

$ sw_vers
ProductName:	Mac OS X
ProductVersion:	10.7.3
BuildVersion:	11D50d

$ java -version
java version "1.6.0_29"
Java(TM) SE Runtime Environment (build 1.6.0_29-b11-402-11D50d)
Java HotSpot(TM) 64-Bit Server VM (build 20.4-b02-402, mixed mode)

Eclipseのインストール

まずはEclipseが必要な訳なので、下記リンクからEclipseを選択してダウンロードします。
（やっぱEasy-linkpostがないとめちゃくちゃ不便だ）

Eclipse Downloads
http://www.eclipse.org/downloads/

ここでちょっと疑問が。Macで選択出来るバージョンが32bitと64bitが選択出来るようなのですが、
このMacはどれなんだろ？Lionは64bitに基本対応しているやつだけのようだけど、Airは64bit対応していないとかいう話もあるし・・・。
なので両方ダウンロードして起動してみた。
どうもどっちも使用出来ているみたいです。アクティビティマネージャで見たら下記の右端がちゃんと32bitと64bitになってた。

Eclipseは基本Javaで出来てるし、作業対象としてはJavaかPHPくらいなもんだとおもうので、ここは64bitでやっといた方がいいのでしょう。
なのでまあ64bitのファイルをダウンロードします。
ADTのページには、Eclipse Classicが推奨とのことなので、Eclipse classicのMacOS64bit版をダウンロードして展開（ダブルクリックで勝手にgzとtarまで展開してくれる）し、出来た「eclipse」フォルダを
移動すればEclipseのインストールは終了。

ADTプラグインのインストール

続いてADTPluginをインストールします。
以前はAndroid SDKのインストールとプラグインのインストールが別になっていた気がしますが、
今回はプラグインをインストールするだけで全部入ってしまうようでした。楽だね。

Eclipseを起動して、Help > Install New Softwareで「Install」画面が出たら、Work Withに下記URLを指定するとネットワークアクセスが始まって
下の表にインストール対象がロードされます。

https://dl-ssl.google.com/android/eclipse/

そしたら下記のような構成になっていると思うので
Developer Tools
・Android DDMS
・Android Developent Tools
・Android Hierarchy Viewer
・Android Traceview

何も考えずすべて（Developer Toolsをチェック）してNext > Next (ライセンス許諾でAccept） > Finishでインストールされます。
今回インストールの際も一度発生したのですが、

HTTP Server 'Bad Gateway' : https://dl-ssl.google.com/android/eclipse/plugins/

みたいなエラーが途中で出たらもう一度やりなおすか、httpsをhttpに変更してやってみましょう。
今回はやり直したらインストール出来たようでした。
インストール終わったら一応Eclipseを再起動して、メニューの「Window」を開いてみてAndroidっぽいメニューが出来てればOKだと思います。
