Subversionのリポジトリ作成にはほぼ鉄則があります。
/svn/svnreposにtestというリポジトリを作成する場合。
さらに、「test」プロジェクトがすでに存在していて、Eclipseを使用してSubvertiveを使っている場合。

svnadmin create /svn/svnrepos/test/
mkdir trunk
mkdir branches
mkdir tags
chown -R apache. test/

これであとはローカルに作成済みである「test」プロジェクトを共用すればOK。
これを見るとなんでもないが、ディレクトリ構成が無名有実というか、ほぼこの構成をもとにSubversionクライアントは動いているようなので、ほぼ確実にこの構成にしないとダメ。
